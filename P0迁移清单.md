# P0迁移清单（simple-nextjs-jwt -> taxonomy）

## 0. 本文目标
- 用最小改动先把源项目的核心业务能力迁入目标项目，并能稳定演示。
- 以“可运行、可验证、可回滚”为第一优先级。

## 1. P0范围（本轮必须完成）
- 认证主链路：注册、登录、当前用户、登出。
- Webhook 主链路：通用接收（含 whisper 分支）+ 消息落库/转发能力（可先降级）。
- Gemini 聊天主链路：基础问答 API（先通再优化）。

> 本次纳入 P0+：统计面板、WebSocket 在线推送、OpenClaw 全部接口、原有前端页面迁移与导航入口。

## 3. 分步执行清单（可打勾）

## 阶段 A：兼容层骨架（1 天）
- [ ] 建立 `app/api/legacy/**` 路由分组，承接源项目接口路径语义。
- [ ] 新建 `lib/legacy/**`，放迁移期适配逻辑（避免污染现有核心模块）。
- [ ] 为 legacy 接口统一响应格式（保留源字段，增加 `success`）。

验收：
- `GET/POST` 路由可达，未实现接口返回明确 `501` 与待办说明。

## 阶段 B：认证链路迁移（1-2 天）
- [x] `register`：接收 email/password，映射到当前用户体系（NextAuth + Prisma）。
- [x] `login`：优先走 NextAuth 机制；提供 legacy 兼容响应（含 user 基本字段）。
- [x] `me`：返回当前会话用户（兼容源字段命名：`is_enabled`, `last_login_at` 等）。
- [x] `logout`：复用现有退出逻辑，清理会话。
- [ ] 提供字段映射层（snake_case <-> camelCase）。

验收：
- Postman/HTTPie 验证四个接口可用，状态码与错误语义符合预期。
- 未登录访问 `me` 返回 `401`。

## 阶段 C：Webhook 主链路迁移（1 天）
- [ ] 实现 `legacy/webhook/receive`：保留 `signature/source/data` 入参语义。
- [ ] 兼容 `source=whisper` 分支：先“入库或日志持久化二选一”，确保不丢消息。
- [ ] 原 `localhost:PORT/api/messages/forward` 内部转发改为：
  - 优先：同进程服务函数调用；
  - 备选：内部 fetch（避免 Node http 直接硬编码端口）。
- [ ] 加入基本幂等键（`source + source_id`）防重复。

验收：
- 模拟 `source=whisper` 请求，返回 200，能查到落库/记录。
- 非 whisper 请求也可正常接收并回显。

## 阶段 D：Gemini API 迁移（0.5-1 天）

## 4. 数据与字段映射（P0最小集）

### 4.1 用户字段
- `id`：保持字符串（目标 Prisma 为 cuid）
- `email`：直接映射
- `is_enabled`：目标暂无同名字段，P0 可固定 `true`（后续在 User 表新增字段）
- `last_login_at`：目标暂无同名字段，P0 可暂存为 `null` 或通过 `updatedAt` 近似

### 4.2 消息字段（whisper）
- `source/source_id/type/title/content/metadata/sender/sender_id/status`
- 若目标库暂无消息表，P0 可先：
  - 方案 A：落到新建 `legacy_messages` 表；
  - 方案 B：落本地文件日志（临时，不建议长期）。

## 6. 风险与处置
- 认证模型不一致（JWT vs NextAuth）
  - 处置：保留 legacy API 外观，内部全部走 NextAuth/Session。
- 字段不一致导致前端兼容问题
  - 处置：建立 DTO 映射层，禁止直接暴露 Prisma 原始对象。
- Webhook 转发链路依赖端口
  - 处置：移除硬编码端口，改服务内调用/内部 fetch。
- 第三方 API 超时
  - 处置：统一超时、重试与错误降级。

---

## 7. 第一批实施顺序（建议）
1. 先做 `legacy/auth/me`（最易验证会话与兼容格式）。
2. 再做 `legacy/auth/login` 与 `legacy/auth/logout`。
3. 然后做 `legacy/webhook/receive`（含 whisper 分支）。
4. 最后做 `legacy/gemini/chat`。
